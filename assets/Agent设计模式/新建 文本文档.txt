Agent 设计模式（九种）

❝
内容来源：微信文章 https://mp.weixin.qq.com/s/tpLjKJ8ZWme0IfHGfk_Xbg（原文以图片/图示为主，请在微信内打开对照阅读。）

本文整理 Agent 常见设计模式，与现有 Agentic 平台中的规划、工具调用、记忆与工作流编排相对应。主流归纳为 **九种** 设计模式。

1. ReAct（推理 + 行动）
思考 → 行动 → 观察，循环迭代。灵活、易调试；LLM 调用多、延迟较高。适合实时交互。
（此处可插入对应配图说明：图：ReAct 循环示意图）

2. Plan-and-Execute（计划 - 执行）
先生成完整计划，再按序执行。可预测、调用少；适应性较弱。适合步骤清晰的长流程。
（此处可插入对应配图说明：图：……）

3. REWOO（无观察推理）
将观察隐式并入下一步执行，消除 ReAct 的串行等待。通常比 ReAct 快 60% 以上；适合批处理。
（此处可插入对应配图说明：图：……）

4. LLM Compiler（DAG 驱动并行）
用有向无环图（DAG）实现工具调用的最大化并行。适合实时看板、高并发工具调用。
（此处可插入对应配图说明：图：……）

5. Basic Reflection（双 Agent 协同）
Generator 生成方案，Reflector 批改并给建议，再迭代修正。师生互动模式。
（此处可插入对应配图说明：图：……）

6. Reflexion（实证驱动进化）
在 Basic Reflection 上增加外部验证与经验沉淀。适用于代码生成、历史错误模式注入。
（此处可插入对应配图说明：图：……）

7. LATS（树搜索决策）
结合蒙特卡洛树搜索（MCTS）与反思机制。适合高价值决策（如投资、医疗方案）。
（此处可插入对应配图说明：图：……）

8. Self-Discover（自适应元模式）
Agent 自主诊断任务特征，动态组装最优模式链。通用 Agent 平台的关键技术。
（此处可插入对应配图说明：图：……）

9. Storm（多专家协同）
融合知识检索、角色扮演与共识生成。用于深度知识生产，可显著降低关键事实错误率。
（此处可插入对应配图说明：图：……）

选型建议（简要）
- 实时交互（客服/游戏）→ ReAct
- 复杂流程（电商大促等）→ Plan-and-Execute
- 批处理任务 → REWOO
- 高并发查询 → LLM Compiler
- 代码/数学验证 → Reflexion
- 战略决策 → LATS
